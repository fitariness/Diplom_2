# Diplom_2 — API-тесты Stellar Burgers

В этой части проверяется API сервиса [Stellar Burgers](https://stellarburgers.education-services.ru/)

Базовый URL: `https://stellarburgers.education-services.ru/api`

Тесты написаны на `pytest` и `requests`, шаги оформлены через `allure-pytest`. Запросы собраны в отдельном API-клиенте, а проверки ответов вынесены в `tests/helpers/checkers.py`

## Зависимости

```bash
pip install -r requirements.txt
```

## Запуск тестов

```bash
pytest
```

## Allure-отчёт

```bash
pytest --alluredir=allure-results
allure serve allure-results
```

## Структура

- `api/` — клиент API
- `config.py` — URL и ожидаемые сообщения
- `tests/` — тесты по функциональности
  - `test_register_user.py` — регистрация
  - `test_login_user.py` — логин
  - `test_create_order.py` — создание заказа
- `tests/helpers/` — генерация данных, работа с пользователем, получение ингредиентов и проверки ответов

## Что проверяется

**Регистрация пользователя**

- создание нового уникального пользователя
- ошибка при повторной регистрации уже существующего пользователя
- ошибка, если не передать `email`
- ошибка, если не передать `password`
- ошибка, если не передать `name`

**Авторизация пользователя**

- вход под существующим пользователем
- ошибка при входе с неверным логином или паролем

**Создание заказа**

- создание заказа авторизованным пользователем с ингредиентами
- создание заказа без авторизации
- создание заказа с валидными ингредиентами
- ошибка при создании заказа без ингредиентов
- ошибка при создании заказа с невалидным hash ингредиента

## Что проверяется в ответах

- HTTP-статус ответа
- поле `success`
- данные пользователя в ответе
- наличие `accessToken` и `refreshToken`
- номер созданного заказа
- текст ошибки для негативных сценариев
